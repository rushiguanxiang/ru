## （Ru-Shi Architecture V2.0）

视觉即物理，观象即执行。

## 第1章 总线与状态基元

### 1.1 八位总线定义

总线宽8位，从左至右（高位至低位）：

| 线号 | bit7 | bit6 | bit5 | bit4 | bit3 | bit2 | bit1 | bit0 |
|------|------|------|------|------|------|------|------|------|
| 名称 | 恒零 | 上爻 | 五爻 | 四爻 | 三爻 | 二爻 | 初爻 | 枢位 |
| 属性 | 阴(0) | 数据线 | 数据线 | 数据线 | 数据线 | 数据线 | 数据线 | 阴阳枢 |

- **bit7（恒零）**：恒为阴，系统接地或片选使能。
- **bit6～bit1（数据线）**：对应六爻，阳为高电平(1)，阴为低电平(0)。
- **bit0（枢位）**：指令脉冲位。本态为阴，易态为阳。

### 1.2 本态与易态（时钟脉冲）

- **本态**：静态寄存器状态，象符无标记。  
  例：䷦（蹇，数据线维持）
- **易态**：在象符右上角标记「一」，代表 bit0 拉高，时钟上升沿触发，指令打入执行。  
  例：䷦一（蹇态被写入/执行）

**任何易象，眼睛一看就知道哪几根线通、哪几根断，以及是否处于执行脉冲。**

## 第2章 数据类型与位域矩阵

### 2.1 阳爻数量即数据类型

《如语言》无需声明 int/float/bool，**阳爻数量天然决定数据属性**：

| 阳爻数 | 类别名 | 物理意义 | 示例 |
|--------|--------|----------|------|
| 0阳 | 休眠态 | 总线清零，断电保持 | ䷁ |
| 1阳 | 标志位 | 布尔值，单线使能 | ䷗（初爻通）|
| 2阳 | 指针态 | 二维寻址坐标（见2.2）| ䷦ |
| 3阳 | 运算态 | ALU核心操作码（见第3章）| ䷾ |
| 4阳 | 强流态 | 块数据/端口吞吐 | ䷡ |
| 5阳 | 冲刺态 | 高优中断/总线仲裁 | ䷪ |
| 6阳 | 满载态 | 全局广播/硬复位 | ䷀ |

### 2.2 二阳位域矩阵（指针寻址网络）

对于恰好两个阳爻的象，其两个高电平位置唯一确定易象。  
用坐标 `(下阳位, 上阳位)` 表示，其中：
- **下阳位**：从初爻数起，第一个阳爻的位置（1～5）
- **上阳位**：从初爻数起，第二个阳爻的位置（2～6）

**位域矩阵（指针路由表）**：

| 上\下 | 1 | 2 | 3 | 4 | 5 |
|-------|---|---|---|---|---|
| 6 | ䷚ | ䷃ | ䷳ | ䷢ | ䷓ |
| 5 | ䷂ | ䷜ | ䷦ | ䷧ | ䷭ |
| 4 | ䷲ | ䷽ | / | / | / |
| 3 | ䷣ | / | / | / | / |
| 2 | ䷒ | / | / | / | / |

> “/”处不属于二阳四阴象（为三阳、四阳等）。

**总线移位操作**：
- **横移（X轴）**：同行右移，移位寄存器操作  
  例：䷚(1,6) ⟷ ䷃(2,6) ⟷ ䷳(3,6) ⟷ ䷢(4,6) ⟷ ䷓(5,6)
- **纵压（Y轴）**：同列下压，地址递减  
  例：䷚(1,6) ↕ ䷂(1,5) ↕ ䷲(1,4) ↕ ䷣(1,3) ↕ ䷒(1,2)
- **正交跃迁（L型）**：一次Y压 + 一次X移，直接命中远端节点  
  例：䷦(3,5) ↕ ䷽(3,4) ⟷ ䷧(2,4)
- **长程闭环**：矩阵中不相邻的节点，通过总线旁路直连（如你的拓扑图中 ䷦ ⇌ ䷽）

**无需画线，坐标即网络。**

## 第3章 三阳运算象（如是ALU指令集）

三阳象共20个，构成《如语言》的核心运算指令（操作码）。三爻皆阳，代表逻辑门电路的深度参与。

| 易象 | 坐标/结构 | 如是指令名 | 物理运算意义 |
|------|-----------|------------|--------------|
| ䷾ | 既济 | **同** | ALU加法/进位 |
| ䷿ | 未济 | **异** | ALU减法/借位 |
| ䷔ | 噬嗑 | **断** | 逻辑与 (AND)，咬合筛选 |
| ䷕ | 贲 | **饰** | 逻辑或 (OR)，装饰附加 |
| ䷞ | 咸 | **感** | 逻辑异或 (XOR)，感应翻转 |
| ䷟ | 恒 | **守** | 数据保持/寄存器暂存 (MOV) |
| ䷩ | 益 | **升** | 算术左移 (乘2) |
| ䷨ | 损 | **降** | 算术右移 (除2) |
| ䷐ | 随 | **入** | 压栈 (PUSH) |
| ䷑ | 蛊 | **出** | 弹栈 (POP) |
| ䷶ | 丰 | **明** | 输出端口写 (OUT) |
| ䷷ | 旅 | **览** | 输入端口读 (IN) |
| ䷴ | 渐 | **步** | 条件跳转 (控制流) |
| ䷵ | 归妹 | **归** | 无条件跳转/子程序调用 (CALL) |
| ䷺ | 涣 | **散** | 清除/归零 (CLR) |
| ䷻ | 节 | **限** | 设置边界/比较 (CMP) |
| ䷼ | 中孚 | **校** | 奇偶校验/总线仲裁 |
| ䷽ | 小过 | **过** | 空操作/时钟节拍等待 (NOP) |

> **指令执行规则**：所有指令必须后跟「一」（易态）才真正执行。  
> 例：`䷞一` 代表执行异或运算。

## 第4章 控制流：消息周期与中断

### 4.1 二十四消息（总线时钟分频）

二十四消息象是 **12个消息象** 的本态与易态按节气顺序排列，构成一个完整的总线电平扫描周期。  
**节气只是外衣，本质是时钟发生器逐级驱动总线。**

按节气时间顺序（从大雪到冬至）排列：

| 节气 | 总线状态 | 节气 | 总线状态 |
|------|----------|------|----------|
| 大雪 | ䷁一 | 芒种 | ䷀一 |
| 小雪 | ䷁ | 小满 | ䷀ |
| 立冬 | ䷖一 | 立夏 | ䷪一 |
| 霜降 | ䷖ | 谷雨 | ䷪ |
| 寒露 | ䷓一 | 清明 | ䷡一 |
| 秋分 | ䷓ | 春分 | ䷡ |
| 白露 | ䷋一 | 惊蛰 | ䷊一 |
| 处暑 | ䷋ | 雨水 | ䷊ |
| 立秋 | ䷠一 | 立春 | ䷒一 |
| 大暑 | ䷠ | 大寒 | ䷒ |
| 小暑 | ䷫一 | 小寒 | ䷗一 |
| 夏至 | ䷫ | 冬至 | ䷗ |

**观察**：
- 阳爻数量从 0（䷁）逐级增至 6（䷀），再逐级减回 1（䷗）
- 易态（带「一」的象）出现在每一步，代表时钟脉冲沿
- 芒种的 ䷀一 是唯一“7个阳”（六阳+枢位），代表溢出中断

### 4.2 中断系统（五阳冲刺与六阳溢出）

- **五阳象（硬件中断）**：如 ䷪一（决断），高优先级外部中断，强制切断当前流。
- **六阳象（溢出复位）**：䷀一，bit0~bit6 全满，触发系统最高级不可屏蔽中断(NMI)，总线全局复位。

## 第5章 《如语言》编程范式（全视觉代码）

### 5.1 语法铁律

- 没有变量声明，**象即地址与数据**。
- 没有分号，**「一」即语句终结与执行**。
- 没有嵌套括号，**矩阵坐标 (X,Y) 与跃迁符 ⟷ ↕ 即控制流**。

### 5.2 代码示例：硬件端口轮询与数据搬移

**需求**：从端口䷢读取数据，若与基准值䷦异或结果非零，则将数据写入䷜，并拉高䷲中断。

**《如语言》写法**：
```plaintext
䷢ 览一     // 端口晋，读入数据，脉冲打入
䷦ 感一     // 与蹇态异或，脉冲打入
䷽ 过一     // 小过空操作，等待标志位稳定
䷜ 饰一     // 若非零，则写至坎态，脉冲打入
䷲ 明一     // 拉高震态，触发中断
```

### 5.3 代码示例：长程闭环推挽

**需求**：实现从蹇(䷦)穿透到小过(䷽)的远端闭环总线赋值。

**《如语言》写法**：
```plaintext
䷦ 守一         // 蹇态锁定，作为起点
[䷦ ⇌ ䷽] 限一  // 声明远端闭环总线，设定边界
䷽ 同一         // 闭环导通，推挽电流涌入
```

### 5.4 代码示例：24节气逐级升频（系统启动时序）

```plaintext
初始：䷁
依次写入以下状态，每个状态持续一个节气时长：
䷁一 → ䷖ → ䷖一 → ䷓ → ䷓一 → ䷋ → ䷋一 → ䷠ → ䷠一 → ䷫ → ䷫一 → ䷀ → ䷀一 → ䷪ → ䷪一 → ䷡ → ䷡一 → ䷊ → ䷊一 → ䷒ → ䷒一 → ䷗ → ䷗一 → ䷗
```

## 结束语：何谓“如是”
  
在《如语言》中：  
- **“如”** 是象的本来面目——总线电平就在那里，不增不减；  
- **“是”** 是枢位的执行——时钟一沿打入，因果决定不虚。

西方的代码，是人对机器的祈使句，需要编译器去猜测、去翻译；  
《如语言》的代码，是人机共用的真像，眼见即总线，手书即通断。

学《如语言》，不是在学一门编程语言，而是在修证硅基宇宙的物理法则。  
**这就是大道至简，这就是如是！**
