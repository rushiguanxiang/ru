# 纳音MMU三十天条

## 概述

纳音MMU（Memory Management Unit）是《如语言》V4.0的内存属性管理核心。三十纳音对应三十条硬件法条，定义内存区域的访问权限、数据宽度、缓存策略与环选择。

## 五行权限法则

```
金(0)：R-- 或 R-X（只读/执行，固化不可改）
木(1)：RW-（可读写，动态生长）
水(2)：RW-（流式FIFO，只进不出或环形）
火(3)：RW-（高速缓存，回写/写通）
土(4)：RW-（静态直写，段基址）
```

## 五音宽度法则

```
宫(0)：8位  商(1)：16位  角(2)：32位  徵(3)：64位  羽(4)：128位
```

## CAM条目结构（32位）

```
[31:27] = 纳音编码（5位，0-29）
[26]    = 环选择（0=周天轮，1=甲子轮）
[25:19] = 基址（7位）
[18:13] = 长度（6位，0-63字）
[12:10] = 权限（3位：R/W/X）
[9:7]   = 五行（3位）
[6:4]   = 五音（3位）
[3:2]   = 策略（2位：0直写,1回写,2写通）
[1]     = 环形标志
[0]     = 有效位
```

## 纳音违例处理

当访问权限与纳音属性冲突时，触发**金之怒**：
- 控制台输出红色"纳音违例"警告
- 触发三级涟漪全局复位
- PC归零，甲子轮回零，纳音MMU重置
