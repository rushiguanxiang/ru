# 万川引擎白皮书

## 摘要

万川引擎是《如语言》V4.0的核心执行单元，采用"场域共振"执行模型替代传统流水线。该引擎以六十四卦象为指令基元，通过纳甲干支寻址、双环存储拓扑、三级涟漪扩散机制，实现同一卦象在不同语境下的多态语义执行。

## 1. 架构概述

### 1.1 设计哲学

西方CPU采用"取指-译码-执行-写回"线性流水线，本质是箭矢思维——有始有终，有去无回。万川引擎采用"同心圆涟漪"执行模型，一象入水，万波俱起，处处响应，无处不是中心。

### 1.2 核心模块

| 模块 | 功能 | 华夏命名 |
|------|------|----------|
| 周天轮 | 128字线性指令存储 | 天之道 |
| 甲子轮 | 60字循环数据缓冲 | 地之道 |
| 四柱锁 | 20位场域字缓存 | 时空锚 |
| 纳音MMU | 30条目CAM属性管理 | 五行司 |
| 全息译码器 | 象码×场域→微操作 | 万川镜 |
| 气交触发器 | bit0检测与脉冲生成 | 乄门 |
| 涟漪扩散器 | 副作用广播网络 | 同心波 |

## 2. 双环存储拓扑

### 2.1 周天轮（主环）

- 深度：128字
- 寻址：7位PC（己寄存器R5[6:0]）
- 循环：PC = (PC + 1) mod 128
- 用途：线性指令流、固定数据、系统状态

### 2.2 甲子轮（辅环）

- 深度：60字
- 寻址：6位环指针（辛寄存器R7[5:0]）
- 循环：Ptr = (Ptr + 1) mod 60
- 用途：DSP滤波、FIFO、音频采样、串口缓冲

### 2.3 双环切换

```
if (纳音.五行 == 水 || 纳音.五行 == 火) → 甲子轮
else → 周天轮
```

切换时机：T2场定节拍（场域字解析后）。

## 3. 场域共振执行模型

### 3.1 五层涟漪结构

```
T1 少阳：取象（从周天轮取指令，甲子轮预取数据）
T2 老阳：场定（解析四柱锁，纳音MMU查询，确定环选择）
T3 少阴：全息译码（象码×场域字→26位微操作码）
T4 老阴：气交（乄门检测，执行微操作，涟漪扩散）
```

### 3.2 20位场域字（四柱锁）

```
[19:18] = Ripple_Level（2位：0=1级,1=2级,2=3级）
[17]    = Ring_Sel（1位）
[16:15] = 保留（2位，羲和模式标志）
[14:11] = DiZhi（4位：0-11）
[10:6]  = NaYin（5位：0-29）
[5:2]   = TianGan（4位：0-9）
[1:0]   = JiaZi_Phase（2位）
```

### 3.3 三级涟漪

| 级别 | 范围 | 触发条件 | 占比 |
|------|------|----------|------|
| 1级 | 单寄存器/单内存 | 普通运算 | 99% |
| 2级 | 相邻槽位/通道 | DMA、缓存预取 | 0.9% |
| 3级 | 全局刷新 | 系统调用、异常 | 0.1% |

### 5.1 译码公式

## 7. 结论

万川引擎证明了易经数理在硅基计算中的可实现性。55,296,000种语义空间、双环拓扑、三级涟漪执行模型，构成了完整的文化计算范式。后续工作包括FPGA原型验证、编译器工具链、以及多核扩展。
